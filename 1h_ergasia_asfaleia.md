# 1η Εργασία — Μελέτη Ασφάλειας Πληροφοριακού Συστήματος

**Πληροφοριακό Σύστημα υπό μελέτη:** Ηλεκτρονικό Κατάστημα (e-shop)

---

## Ερώτημα 1: Καταγραφή του υπό μελέτη συστήματος

### 1.1 Γενική περιγραφή

Το υπό μελέτη Πληροφοριακό Σύστημα (ΠΣ) είναι ένα μικρού μεγέθους ηλεκτρονικό κατάστημα (e-shop) λιανικής πώλησης προϊόντων, το οποίο παρέχει on-line υπηρεσίες σε εγγεγραμμένους και μη εγγεγραμμένους χρήστες μέσω web διεπαφής. Το σύστημα φιλοξενείται σε ιδιόκτητη υποδομή και διασυνδέεται με εξωτερική υπηρεσία διεκπεραίωσης πληρωμών (συνεργαζόμενη τράπεζα, 3rd party). Ο αριθμός των εγγεγραμμένων χρηστών ανέρχεται σήμερα σε περίπου 1.000.

### 1.2 Παρεχόμενες υπηρεσίες

Το σύστημα παρέχει τις ακόλουθες on-line υπηρεσίες:

**Υ1. Αναζήτηση προϊόντων (Product Search)**
Παρέχεται η δυνατότητα σε όλους τους επισκέπτες (εγγεγραμμένους ή μη) να αναζητούν προϊόντα στον κατάλογο του καταστήματος. Η αναζήτηση πραγματοποιείται είτε μέσω πεδίου ελεύθερης αναζήτησης (full-text search) είτε μέσω πλοήγησης σε ιεραρχικά μενού κατηγοριών και υποκατηγοριών. Τα αποτελέσματα αντλούνται από τη βάση δεδομένων προϊόντων και παρουσιάζονται μέσω της web εφαρμογής.

**Υ2. Εγγραφή και διαχείριση χρηστών (User Registration & Account Management)**
Οι νέοι χρήστες εγγράφονται μέσω web φόρμας, παρέχοντας στοιχεία ταυτοποίησης και επικοινωνίας (ονοματεπώνυμο, διεύθυνση αποστολής, e-mail, τηλέφωνο). Κατά την εγγραφή δημιουργούν μοναδικό κωδικό πρόσβασης, ο οποίος ανανεώνεται υποχρεωτικά κάθε 2 έτη το αργότερο. Τα στοιχεία των χρηστών αποθηκεύονται στη βάση δεδομένων του συστήματος.

**Υ3. Ηλεκτρονικές αγορές (On-line Payments)**
Οι εγγεγραμμένοι χρήστες πραγματοποιούν ηλεκτρονικές αγορές προϊόντων μέσω της εφαρμογής. Πριν από την πρώτη αγορά, ο χρήστης εισάγει τα στοιχεία της πιστωτικής/χρεωστικής του κάρτας, τα οποία αποθηκεύονται στη βάση δεδομένων σε κρυπτογραφημένη μορφή. Η μετάδοση των στοιχείων κάρτας προστατεύεται με χρήση πρωτοκόλλου SSL/TLS. Η τελική διεκπεραίωση της πληρωμής γίνεται μέσω συνεργαζόμενης τράπεζας ως 3rd party υπηρεσία, στην οποία ανακατευθύνεται ο χρήστης.

### 1.3 Αρχιτεκτονική συστήματος

Η αρχιτεκτονική του δικτύου ακολουθεί κλασικό τριών επιπέδων (3-tier) σχήμα, με διαχωρισμό σε ζώνες ασφάλειας μέσω firewall:

```
                      ┌──────────────────────┐
   Internet ────────► │   External Firewall  │
                      └──────────┬───────────┘
                                 │
                          ┌──────▼───────┐
                          │     DMZ      │
                          │  Web Server  │ ◄─── Apache httpd 2.4.60
                          │ (Apache+PHP) │      PHP 8.2.15
                          └──────┬───────┘
                                 │
                      ┌──────────▼──────────┐
                      │  Internal Firewall  │
                      └──────────┬──────────┘
                                 │
                          ┌──────▼───────┐
                          │  Internal    │
                          │   Network    │
                          │              │
                          │  ┌────────┐  │
                          │  │   DB   │  │ ◄─── MySQL 8.1.0
                          │  │ Server │  │
                          │  └────────┘  │
                          │              │
                          │  ┌────────┐  │
                          │  │ Admin  │  │ ◄─── Διαχειριστής
                          │  │   PC   │  │
                          │  └────────┘  │
                          └──────────────┘

                ╔════════════════════════════╗
                ║  3rd Party Payment Gateway ║ ◄── HTTPS/TLS
                ║   (Συνεργαζόμενη Τράπεζα)  │
                ╚════════════════════════════╝
```

## Ερώτημα 3: Αντιστοίχιση υπηρεσιών και υπολογιστικών συστημάτων

Στον ακόλουθο πίνακα παρουσιάζεται η αντιστοίχιση μεταξύ των τριών παρεχόμενων υπηρεσιών και των τριών υπολογιστικών συστημάτων που τις υποστηρίζουν. Ένα υπολογιστικό σύστημα μπορεί να συμμετέχει στην παροχή περισσότερων υπηρεσιών.

| Υπηρεσία | Web/App Server | Database Server | Administrator PC |
|----------|:--------------:|:---------------:|:----------------:|
| **Υ1. Αναζήτηση προϊόντων** | ✓ | ✓ | ✓ (έμμεσα — διαχείριση καταλόγου) |
| **Υ2. Εγγραφή & διαχείριση χρηστών** | ✓ | ✓ | ✓ (έμμεσα — διαχείριση λογαριασμών) |
| **Υ3. Ηλεκτρονικές αγορές** | ✓ | ✓ | ✓ (έμμεσα — επιβεβαίωση/υποστήριξη παραγγελιών) |

### Επεξήγηση αντιστοίχισης

**Υ1 — Αναζήτηση προϊόντων**
- *Web/App Server:* Δέχεται το αίτημα αναζήτησης μέσω HTTP(S), εκτελεί τη λογική της εφαρμογής (PHP), συνθέτει το ερώτημα και παρουσιάζει τα αποτελέσματα.
- *Database Server:* Εκτελεί τα SQL ερωτήματα στον πίνακα προϊόντων και επιστρέφει τα αποτελέσματα.
- *Administrator PC:* Συμμετέχει έμμεσα μέσω της ενημέρωσης/διαχείρισης του καταλόγου προϊόντων (προσθήκες, διορθώσεις).

**Υ2 — Εγγραφή & διαχείριση χρηστών**
- *Web/App Server:* Παρέχει τη φόρμα εγγραφής, επικυρώνει τα δεδομένα εισόδου (validation), διαχειρίζεται sessions και authentication.
- *Database Server:* Αποθηκεύει τα στοιχεία των χρηστών και τα hashed passwords, υποστηρίζει ενημερώσεις/διαγραφές.
- *Administrator PC:* Έμμεση συμμετοχή για διαχειριστικές ενέργειες (επαναφορά κωδικών, απενεργοποίηση λογαριασμών, υποστήριξη χρηστών).

**Υ3 — Ηλεκτρονικές αγορές**
- *Web/App Server:* Διαχειρίζεται το καλάθι αγορών, την επικοινωνία με το payment gateway της τράπεζας μέσω TLS, την επιβεβαίωση παραγγελίας.
- *Database Server:* Αποθηκεύει τις παραγγελίες, το κρυπτογραφημένο PAN της κάρτας, το ιστορικό συναλλαγών.
- *Administrator PC:* Έμμεση συμμετοχή για επιβεβαίωση/επεξεργασία παραγγελιών, χειρισμό επιστροφών, εξυπηρέτηση πελατών.

> **Σημείωση:** Η εξωτερική υπηρεσία διεκπεραίωσης πληρωμών (payment gateway της συνεργαζόμενης τράπεζας) είναι 3rd party και δεν συμπεριλαμβάνεται στο μοντέλο αγαθών του υπό μελέτη ΠΣ, καθώς δεν τελεί υπό τη διαχείριση του οργανισμού.
